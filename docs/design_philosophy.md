# LexPulse — 設計精神與亮點說明

> 本文件說明設計理念、與 project3.md 規格的對應關係，以及我們在規格基礎之上額外延伸的設計決策與亮點功能。

## 一、project3.md 要求了什麼

project3.md（「七分鐘衝刺閱讀器與情境式銜接機制」）定義了三件事：

### 1. 衝刺前啟動畫面
> 在計時器開始前，先顯示「原始文件標題、領域標籤、行動呼籲（7 分鐘 / 5 卡 / 隨即測驗）」

### 2. 可滑動的閱讀介面
> 行動優先、類 Instagram 限時動態風格；頂部顯示進度（N/5）與倒數計時；計時器與 Visibility API 綁定

### 3. 交接畫面 + 橋接資料表
> 計時器歸零後鎖定畫面，顯示「衝刺完成，正在生成你的測驗⋯⋯」，並透過 `LearningJourney_Map` 把 Sprint Session 與 Quiz Session 1:1 對應，為後續 AI 分析鋪路

這三點是規格的**核心底線（baseline）**。

## 三、超越規格的設計：我們多做了什麼

### 3.1 SM-2 間隔重複引擎（完整實作，非概念演示）

project3.md 的上游規格（microlearning_app_spec.md）只提到「使用 SM-2 或同等演算法」並給出虛擬碼。我們不只寫了虛擬碼，而是完整實作並接入學習流程：

- **品質分數映射**：`score_to_quality()` 將測驗分數（0-100）轉換為 SM-2 的 quality（0-5）
- **每人每概念獨立追蹤**：`review_schedule` 表以 `(user_id, concept_tag)` 為 UNIQUE KEY，精細到標籤級別
- **強化場次降低 XP**：複習場次（review）的 XP Tier 表刻意設為學習場次的一半，鼓勵首次學習、避免靠刷複習衝 XP
- **Dashboard 整合**：`next_review_at <= today` 的項目直接顯示在儀表板「待複習」清單

```
正確率 90-100% → quality 5 → interval 指數成長（已掌握）
正確率 < 40%   → quality 0 → interval 重設為 1（需要立即重學）
```

### 3.2 XP + 六階等級系統（完整遊戲化迴路）

規格只提到「等級/掌握度」概念，我們建構了完整的遊戲化正向迴路：

**六個等級** 對應金融法遵職涯語境（而非通用「Level 1, 2, 3」）：

| 等級 | 名稱 | XP 門檻 |
|------|------|---------|
| 第 1 階 | 新手法遵員 | 0 XP |
| 第 2 階 | 合規見習生 | 100 XP |
| 第 3 階 | 法遵分析師 | 300 XP |
| 第 4 階 | 合規達人 | 700 XP |
| 第 5 階 | 法遵大師 | 1,500 XP |
| 第 6 階 | 法遵守護者 | 3,000 XP |

**升級觸發鏈**（讓每次升級都有儀式感）：
1. `update_user_xp()` 偵測升級事件
2. 前端播放彩紙特效（CSS confetti animation）
3. 吉祥物泡泡對話框出現
4. 彈出「升級獎勵」選擇面板 → 使用者選擇 KGI 點數 或 學習時數
5. `UNIQUE(user_id, level_reached)` + `INSERT IGNORE` 確保每個等級只能領一次

**XP 差異化設計**：
- 學習場次（首次）XP Tier：[20, 30, 45, 65, 90, 125]
- 複習場次 XP Tier：[10, 15, 22, 32, 45, 60]
- 第一次作答 vs 修改答案分開追蹤（`first_attempt_map`），SM-2 以「首次作答」為品質依據，杜絕作弊刷分

### 3.3 連續學習天數 + 凍結券機制

規格提到「streak 指標」但未說明機制。我們設計了：

- **連續天數**：從今天往回計算，連續有 quiz 完成記錄的天數
- **凍結券（Streak Freeze）**：每累積 7 個連續天，自動獲得 1 張凍結券
- **凍結券使用**：允許中間斷 1 天而不歸零，模擬現實中的「偶爾缺席」
- **Dashboard 視覺化**：火焰圖示 + 凍結券數量，明確顯示可用緩衝次數

這個設計直接借鑒了 Duolingo 的核心留存機制，在金融法遵訓練情境下同樣適用。

### 3.4 三幣獎勵生態（LexPulse 獨有設計）

這是整個系統最具差異化的設計。我們建構了一個完整的內部貨幣生態：

```
學習時數 (learning_hours)
    ↑ 來源: 完成衝刺（自動計入，秒數/3600）
    ↑ 來源: 升級選擇獎勵
    ↓ 用途: 換成 KGI 點數（1小時 = 50點）

KGI 點數 (kgi_points)
    ↑ 來源: 學習時數兌換
    ↑ 來源: 升級選擇獎勵
    ↓ 用途: 兌換商品目錄商品

考績時數 (perf_hours)
    ↑ 來源: 兌換商品目錄中「考績時數」類商品
    ↓ 用途: 計入個人年度考績學習時數（實際業務意義）
```

**為什麼這樣設計？**

- `learning_hours` 是學習行為的直接度量，讓學員「看見」時間投入的價值
- `kgi_points` 是流通媒介，讓學習時間可以換成有形獎勵
- `perf_hours` 對應真實業務場景：金融業合規訓練時數可計入考績，這個轉換讓學員有強烈的外部動機

**交易稽核日誌**：所有貨幣異動（賺取、兌換、兌現）均寫入 `reward_transactions`，具備完整的 `txn_type`、`points_delta`、`hours_delta`、`note` 欄位，滿足合規稽核需求。

### 3.5 學習時數自動計入（衝刺結束自動觸發）

在初版設計中，`learning_hours` 只靠升級獎勵補充，造成餘額長期為 0 的體驗斷層。我們在 `end_sprint()` 中新增自動計入邏輯：

```python
learn_hrs = round(elapsed / 3600, 2)   # 7分鐘 ≈ 0.12小時
if learn_hrs > 0:
    UPDATE users SET learning_hours = learning_hours + learn_hrs
    INSERT INTO reward_transactions (earn_hours_study, ...)
```

每次學習衝刺結束後，學員立即看到錢包數字增加，**行為→回饋的閉環時間壓縮到秒級**。

### 3.6 完整的法遵知識庫（四個真實資料來源）

規格沒有要求搜尋功能，這完全是我們主動加入的功能層：

- **裁罰案例資料庫**：機構名稱、案由、裁罰內容（全文索引）
- **金融法規資料庫**：法規類別、標題、全文
- **全國法規資料庫**：法條層級、條號、條文（JSONL 格式匯入）
- **金管會函令資料集**：發文字號、主旨、全文

設計亮點：
- `FULLTEXT INDEX` 支援中文關鍵字全文搜尋
- 動態篩選器：不同資料類型有不同篩選維度（機構、法規體系、類別）
- 分頁設計：每頁 20 筆，支援大資料量
- 單筆詳情彈窗：長文本欄位特殊處理（不折行截斷，保留格式）

**這個功能讓 LexPulse 從「學習平台」升維到「知識工作台」**，學員不只在系統內學習，還可以在工作中即時查詢法規依據。

### 3.7 幂等資料庫遷移系統

規格假設資料庫一次建立完成。實際 POC 開發中，需求持續演化，我們設計了一個不需要手動介入的遷移系統：

```python
# migrations.py 節錄
def _add_col_if_missing(cur, table, column, col_def):
    cur.execute(
        "SELECT COUNT(*) FROM information_schema.COLUMNS "
        "WHERE TABLE_SCHEMA=%s AND TABLE_NAME=%s AND COLUMN_NAME=%s",
        (DB_NAME, table, column)
    )
    if cur.fetchone()["COUNT(*)"] == 0:
        cur.execute(f"ALTER TABLE {table} ADD COLUMN {column} {col_def}")
```

- 每次啟動自動執行，有欄位才加，沒有不重複加
- 結構演化不需要 downtime，不需要人工介入
- 新模組（第 13-18 個）也透過 migration 植入，確保更新後舊環境自動補齊

## 四、設計精神總結

### 「學習行為立即可見」

從衝刺結束的瞬間，學習時數就出現在錢包。XP 在提交測驗後立即更新。這不是裝飾，是**行為設計**：讓回饋閉環的延遲趨近於零，強化學習行為的神經迴路。

### 「每個數字都有現實意義」

- 學習時數 → 反映真實投入時間（分鐘 / 3600 = 小時）
- 考績時數 → 對應年度考績學習記錄（台灣金融業合規要求）
- 連續天數 → 每天的習慣積累
- SM-2 間隔 → 科學記憶曲線的數學表達

沒有一個數字是純粹裝飾性的。

### 「合規即場景」

法遵知識庫的存在讓系統超越了「訓練工具」的定位。學員在完成一個「洗錢防制法規」模組後，可以立即在同一個平台搜尋真實的 AML 裁罰案例。學習脈絡與工作脈絡無縫銜接。

### 「為 AI 分析預留接口」

`LearningJourney_Map` 把衝刺行為（閱讀時間、切換分頁次數）與測驗表現（分數、回答速度）一一對應。未來任何 ML 模型都可以從這張表出發，分析「閱讀行為特徵 → 測驗表現」的預測關係，無需更動資料庫結構。

### 「摩擦越少，行為越多」

- 不需要 App 安裝（純 Web）
- 7 分鐘上限讓使用者永遠知道「這次要花多久」
- 無強制序列（可任選模組）
- 凍結券讓「偶爾缺席」不等於「功虧一簣」

每一個設計決策都是在降低啟動阻力，提高完成率。
